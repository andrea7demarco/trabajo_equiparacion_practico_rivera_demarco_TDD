# Introducción a la práctica

El presente trabajo tiene como objetivo abordar de manera práctica la aplicación de TDD mediante el desarrollo del backend de un sistema de agenda para la gestión de turnos. El sistema fue implementado utilizando el lenguaje de programación C#, poniendo especial énfasis en la correcta aplicación de la metodología TDD por sobre la complejidad funcional del sistema. De esta forma, el foco del proyecto no se encuentra en la construcción de un producto completo listo para producción, sino en el análisis del impacto que tiene TDD sobre el proceso de desarrollo y el diseño del software resultante.

Durante el desarrollo del proyecto se documentó de manera sistemática el proceso de trabajo, incluyendo las iteraciones del ciclo **Red–Green–Refactor**, las decisiones de diseño adoptadas, la experiencia de programación con TDD y las conclusiones obtenidas a partir de la práctica. Toda esta documentación se encuentra disponible y versionada en el mismo repositorio del proyecto, con el objetivo de ofrecer una visión clara y transparente del proceso seguido, así como facilitar su análisis posterior.

Finalmente, el trabajo se limita deliberadamente al desarrollo del backend del sistema bajo una arquitectura monolítica, excluyendo aspectos como la interfaz de usuario, la persistencia en bases de datos y otras funcionalidades propias de un sistema que pretende ser de utilidad en un entorno de producción. Esta decisión permitió concentrar los esfuerzos en la aplicación disciplinada de TDD y en el análisis de su influencia sobre el diseño, la detección temprana de errores, la refactorización del código y nuestra experiencia como programadores durante el proceso de desarrollo.
