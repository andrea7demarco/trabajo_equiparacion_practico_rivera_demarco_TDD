## 2 – Objetivos de TDD
Esta metodología tiene una serie de objetivos que abarcan tanto aspectos técnicos del software como tiempo del proceso de desarrollo.
Uno de sus objetivos centrales, es mejorar el diseño interno del software. Desde el punto de vista metodológico, TDD busca favorecer estructuras simples, mantenibles y fácilmente extensibles, de modo que el sistema pueda escalar de forma controlada.
Un segundo objetivo fundamental es la detección temprana de errores. Su ejecución frecuente de pruebas permite identificar fallos inmediatamente después de introducirlos, reduciendo el costo de corrección y evitando la propagación de errores hacia etapas posteriores del desarrollo, donde el impacto técnico y organizacional de los errores es significativamente menor.
Finalmente, otro objetivo relevante de TDD es usar las pruebas como “documentación viva” del sistema. Las pruebas escritas describen el comportamiento esperado de las unidades de código en forma ejecutable y verificable. Al ver estas pruebas, sirven como ejemplos claros de uso, facilitando su entendimiento.
